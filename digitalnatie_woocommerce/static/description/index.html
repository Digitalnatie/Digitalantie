WooCommerce Connector

Bi-directional sync between WooCommerce and Odoo 18 Community. Products, customers, orders, stock and prices — kept in step in real time, with webhooks, multi-instance support and full meta-field mapping.

Odoo 18.0 Community WooCommerce REST v3 Multi-Instance Webhooks No SaaS dependency

Why this connector

Built by an Odoo integrator for production WooCommerce ↔ Odoo workflows. Sane defaults, predictable behaviour, no surprises.

⇄

Bi-directional, fine-grained

Per-instance, per-object direction control: disabled, import, export, or both. Odoo stays authoritative for stock and prices unless you explicitly override.

⚡

Instant via webhooks

Signed HMAC-SHA256 webhooks for orders, products and customers. One-click registration creates them inside WooCommerce.

🏬

Multi-instance

Connect as many WooCommerce stores as you need. Each cron iterates every active store and isolates errors.

🛡️

Safe by default

All cron jobs are inactive on install. Stock recovery imports are gated by an explicit confirmation. SSL verified by default.

📜

Full sync logs

Every operation logged with level, WooCommerce reference and Odoo target. Daily cleanup with configurable retention.

🧩

Custom field mapping

Map any WooCommerce meta_data entry to any Odoo field on products, variants, customers or orders.

What syncs

Everything you need for a typical eCommerce ↔ ERP integration.

WooCommerce → Odoo

  • Products and variants (categories, images, taxes)
  • Customers (billing & shipping addresses)
  • Orders (lines, shipping, fees, discounts, taxes)
  • Stock recovery import (manual)
  • Price recovery import (manual)
  • Custom meta fields

Odoo → WooCommerce

  • Products, prices, descriptions, images
  • Stock levels (Odoo is the source of truth)
  • Categories
  • Order fulfillment status updates
  • Custom Odoo field values → meta_data
  • Full and partial refunds via wizard

Installation in 5 steps

1

Drop the module into your addons folder

Copy digitalnatie_woocommerce into any folder declared in your addons_path.

2

Update apps list & install

Activate developer mode, Apps → Update Apps List, then install WooCommerce Connector.

3

Generate WooCommerce REST keys

WooCommerce → Settings → Advanced → REST API → Add Key (Read/Write).

4

Create your first instance in Odoo

WooCommerce → Instances → New. Fill in URL + keys + warehouse, click Test Connection.

5

(Optional) Register webhooks and enable crons

Click Register Webhooks on the instance form. Enable the cron jobs you want under Settings → Technical → Scheduled Actions.

FAQ

Does this require a license server or activation key?

No. The module is fully self-contained. No telemetry, no SaaS, no key server. Standard OPL-1 conditions apply.

Will scheduled syncs overwrite my Odoo stock?

Never. Scheduled and manual Export Inventory always push Odoo → Woo. To pull stock back, the explicit Import Stock (Recovery) action is gated by a confirmation dialog.

Multiple WooCommerce stores?

Yes. Create one instance per store. Crons iterate every active instance and isolate errors.

Which WooCommerce versions?

Any version exposing the wc/v3 REST API — WooCommerce 3.5 and later.
