Getting started

Up and running in 5 steps

1

Drop the module in your addons folder

Copy digitalnatie_woocommerce into any directory declared in your addons_path.

2

Update apps list & install

Activate developer mode → Apps → Update Apps List → install WooCommerce Connector.

3

Generate WooCommerce REST API keys

WooCommerce → Settings → Advanced → REST API → Add Key → Read/Write → copy Consumer Key & Secret.

4

Create your first instance in Odoo

WooCommerce → Instances → New. Enter URL, keys and warehouse, then click Test Connection.

5

(Optional) Register webhooks & activate crons

Click Register Webhooks on the instance form. Enable the crons you need under Settings → Technical → Scheduled Actions.

FAQ

Common questions

Does this require a license server or activation key?
No. The module is fully self-contained. No telemetry, no SaaS, no key server. Standard OPL-1 conditions apply.
Will scheduled syncs overwrite my Odoo stock?
Never. Scheduled and manual Export Inventory always pushes Odoo → WooCommerce. To pull stock back, use the explicit Import Stock (Recovery) action — it is gated by a confirmation dialog.
Can I connect multiple WooCommerce stores?
Yes. Create one instance record per store. Crons iterate every active instance and isolate errors so one failing store never blocks the others.
Which WooCommerce versions are supported?
Any version exposing the wc/v3 REST API — WooCommerce 3.5 and later.
How are webhooks secured?
Webhooks are validated with HMAC-SHA256 against a per-instance auto-generated secret. Invalid signatures return HTTP 401 without revealing whether the instance exists.
Troubleshooting

Quick diagnostics

Connection test fails
Check API Base URL and keys on the instance form. Re-click Test Connection.
Webhook not firing
WooCommerce → Settings → Advanced → Webhooks → Delivery Log.
Sync error
WooCommerce → Sync Logs → filter by Level = Error for full details.
SSL error on staging
Instance form → uncheck Verify SSL (development only, never in production).
Cron not running
Settings → Technical → Scheduled Actions → find the cron and tick Active.
Log table growing large
Settings → WooCommerce → reduce the Log Retention period.
Security

Built with security in mind

  • Per-store Consumer Key / Secret stored in the database, restricted to Odoo administrators via ir.model.access.csv.
  • Webhook payloads validated with HMAC-SHA256 against a per-instance auto-generated secret.
  • Invalid webhook signatures return HTTP 401 without revealing whether the instance exists.
  • SSL verification enabled by default. Only documented to be disabled for development environments.
  • No outbound calls to any server other than the WooCommerce REST endpoint you configure. Zero telemetry.
  • Per-cron commit/rollback isolation: a poisoned instance cannot corrupt the others.
